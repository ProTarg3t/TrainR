SYSTEM

← PROTOTYPE

Athletic monochroom met één signaalkleur. Mono accents voor data en labels, Inter voor body & headings. Doel: leesbaarheid en focus boven decoratie. Strakke, militair-functionele toon.

01 — Design principles

P.01

Sessie is heilig

Tijdens een workout is het scherm één instrument. Geen secundaire CTAs, geen meldingen, geen scroll-druk.

P.02

Eén signaal

Signal yellow markeert state-changes en de actieve set. Andere kleuren zijn status, niet decoratie.

P.03

Numeriek = mono

Tijden, sessies, percentages, streaks: altijd Geist Mono met tabular-nums. Body & koppen blijven Inter.

P.04

Density is een keuze

Compact / Cozy / Comfortable als first-class token. Default = Cozy. Geen gratis whitespace, geen claustrofobie.

07 — Data schema (voor Claude Code)

Concrete TypeScript-achtige interfaces om mee te beginnen. Slaat lokaal op (IndexedDB / Dexie / Drizzle SQLite). Sync naar backend optioneel later.

UserProfile

interface UserProfile {
  id: string;
  name: string;
  dob: string;                     // ISO date — leeftijd afgeleid
  sex: 'm' | 'f' | 'x' | '-';      // beïnvloedt BMR
  height_cm: number;
  weight_kg: number;               // huidige gewicht
  start_weight_kg: number;         // bij onboarding vastgelegd
  target_weight_kg: number | null; // doel — null = geen target
  goal: 'lose' | 'maintain' | 'muscle' | 'strength';
  level: 'beginner' | 'inter' | 'pro';
  units: 'metric' | 'imperial';    // alleen displayvoorkeur
  notifications: boolean;
  created_at: string;
  updated_at: string;
}

// Afgeleide waardes (computed, niet opgeslagen):
// age          = floor((now - dob) / year)
// bmi          = weight_kg / (height_cm/100)^2
// bmr (M-SJ)   = 10*weight + 6.25*height - 5*age + (sex=='m' ? 5 : -161)
// tdee         = bmr * activity_factor (1.375 sedentary → 1.725 active)

Routine + Exercise

type Muscle =
  | 'chest' | 'shoulders' | 'biceps' | 'triceps' | 'forearms'
  | 'abs' | 'hipflexors' | 'quads' | 'hamstrings' | 'glutes'
  | 'calves' | 'back' | 'lowback' | 'traps';

interface Exercise {
  id: string;
  name: string;                    // "BURPEES"
  category: 'core' | 'cardio' | 'push' | 'pull' | 'legs' | 'full';
  instruction: string;             // 1 zin, max 80 tekens
  tip?: string;                    // optionele coach-tip (alleen detail-scherm)
  met: number;                     // MET-waarde voor calorieberekening
                                   //   bv. burpees=8.0, jumping_jacks=8.0,
                                   //   walking=3.5, squats=5.0
  heat: Partial<Record<Muscle, 0|1|2|3|4>>;
  default_secs?: number;           // tijdgebaseerde oefening
  default_reps?: number;           // rep-gebaseerde oefening
}

interface RoutineExercise {
  exercise_id: string;
  secs?: number;                   // override per routine
  reps?: number;
  rest_secs: number;               // standaard 5–15
}

interface Routine {
  id: string;
  name: string;                    // "Cardio Burn"
  category: Exercise['category'];  // primaire categorie
  exercises: RoutineExercise[];
  created_at: string;
  is_built_in: boolean;            // standaard-routines vs user-made
}

Session log (achteraf)

interface Session {
  id: string;
  routine_id: string;
  routine_name_snapshot: string;   // ivm latere routine-edits
  started_at: string;
  finished_at: string;
  duration_s: number;              // werkelijk getraind
  exercises_done: number;
  exercises_skipped: number;
  calories_kcal: number;           // berekend, zie hieronder
  rpe: number | null;              // 1–10, null = niet ingevuld
  heat_summary: Record<Muscle, 0|1|2|3|4>; // max heat per spier
}

// Calorie-formule per oefening (en gesommeerd):
//   kcal = MET × weight_kg × (duration_s / 3600)
//
// Voor een hele sessie:
//   total_kcal = Σ exercises ( MET_i × weight_kg × secs_i / 3600 )
//
// Voorbeeld: 70kg, burpees (MET 8.0), 40s
//   8.0 × 70 × (40/3600) = 6.2 kcal

08 — Advies om te imporeren in Claude Code

  1. Drop tokens.css als root in /styles/ — gebruik de CSS variabelen overal. Geen hardcoded hex.
  2. Type-pairing vastleggen: Inter (400/500/600/700) voor alles wat tekst is, Geist Mono (400/500/600) voor data, labels, eyebrows, timers. Configureer in tailwind.config.ts als fontFamily.sans en fontFamily.mono.
  3. Vervang de oranje/paars dual-accent door één --signal. Categorieën (CORE, CARDIO, PUSH) blijven gekleurd via --status-*, niet als brand.
  4. Kill mono voor body: oude screens hadden alles in mono → onleesbaar. Mono alleen voor: tabs, eyebrows, numerics, kleine meta (4 OEF · 12:30), en cijfers in het timer-oog. Headings en instructies altijd Inter.
  5. Body-figuur: vervang de gestockte 3D illustratie door BodyMap (zie body-map.jsx) — abstracte regio's met heatmap fill 0–4. Schaalbaar, datadriven, geen assets nodig.
  6. Active workout = coach mode: timer-ring (260px) + oefening-naam (Inter 700, 28px) + 1-zin instructie + body heat preview + volgende oefening kaartje + 2-knop control row. Geen scrolling tijdens werken, alles past in 1 viewport.
  7. Density token: data-density attribuut op <html> → 3 niveaus. Gebruik --space-row, --card-pad, --gutter in plaats van magic numbers.
  8. 4-tab nav: HOME / ROUTINES / STATS / PROFILE. Stats placeholder klaarzetten (lege state met "Begin je eerste sessie") tot grafieken er komen.
  9. Copy in NL, militair-strak: PROTOCOL, EXECUTE, SESSIE, RUST, SLA OVER, HERVAT. Geen emoji, geen "Goed bezig!".
  10. Motion guidelines: --dur-fast voor hover/press, --dur-base voor transitions, --dur-slow voor screen changes. Easing altijd --ease-out tenzij heel-en-weer.
